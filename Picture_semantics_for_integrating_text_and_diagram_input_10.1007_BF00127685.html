Abstract. The saying 'a picture is worth a thousand words' exemplifies the great value of pictures in describing a scenario. Pictures convey spatial information in a compact form, allowing textual descriptions to concentrate on the non-spatial (henceforth, contextual) properties of objects. The difficult task in integrating text and diagrammatic input to a system is to establish coreference - matching object references in the text to objects in the diagram.
We show that the coreference problem can be greatly simplified if limited contextual information can be provided directly in diagrams. We present a methodology, the Picture Semantics description language, for associating contextual information with objects drawn through graphical editors. Then, we describe our implemented research tool, the Figure Understander, which uses this methodology to integrate the differing information in text and graphically-drawn diagrammatic input into a single unified knowledge base description. We illustrate the utility of our methods through examples from two independent domains.
Key words: picture semantics, coreference, diagram and text integration, spatial reasoning

1. Introduction

Maps, schematic diagrams, and pictures accompanying stories are three examples of the many ways in which pictorial information is used in human communication. Larkin and Simon describe the benefits of using both diagrams and text to describe a scene in their classic paper, "Why a Diagram is (Sometimes) Worth 10,000 Words" (Larkin et al., 1987). Pictures can be used to compactly describe the spatial properties of objects in a scene. This allows any accompanying textual descriptions to concentrate on the non-spatial domain-specific and problem-specific (henceforth, contextual) properties of objects. To clarify our terminology, the rectangular object in the diagram of Figure 1 has the spatial property that it is rectangular, and the contextual property that it is a magnetic field.

1.1. The coreference problem

Diagrams and text typically describe the same object in different ways. Figure 1 shows the diagram and text statement for a Physics problem (Resnick et

Each form of input, diagram and text, contains unique information. In Figure 1, the diagram shows the position and shape of the circular object, but it is the word 'disk' in the text that reveals that it is a solid object, and it is the word 'copper' that reveals that it is a conducting object.

The coreference problem (Novak et al., 1993) is to associate object references in text with the appropriate diagram objects. Solutions to the coreference problem are important because the unique information in each form of input, diagrams and text, can be integrated only after the mappings between diagram and textual objects are identified.
To solve the coreference problem, human readers can make use of a large "commonsense" knowledge base to quickly visualize the spatial properties of an object mentioned in a text description and identify the appropriate object(s) in a picture. Similar strategies have been adopted in previous work to automate the ability to solve the coreference problem. One approach is to use model libraries which map domain-specific terms to a spatial model for the object (Novak et al., 1993), such as the model that a 'car' is a rectangle with four circles attached to it. Another approach is to use an image database to directly associate a picture with a textual term (Rowe et al., 1993; Srihari, 1994).
Both approaches face problems with the completeness of the model library or image database and ambiguities in assessing the quality of a match against a given input. Unfortunately, a given object or collection of objects in a diagram may map to several models in a model library, or objects in the input may have a different shape or orientation, and thus appear to be different, than objects in an image database.
For example, in interpreting the diagram in Figure 1, a human reader can easily recognize that the sloping portion of the triangular base forms the 'track', that the vertical piece forms the support for the track, and that the horizontal line is the ground. In a model library, a possible mapping for the word 'track' is that it is a line object in a diagram.
In the text of Figure 1, there is only one reference to a 'line' object - the track. The diagram contains three line objects - the track, the support, and the ground. In this case, we would either need a model for a 'supported track' in the library which would include all three line objects in the diagram, or the underlying system would have to perform complex reasoning about the information in the text to associate only the sloping line with the word 'track'.

1.2. Overview of our goals and solution

In previous work, researchers have addressed the problem of automatically interpreting text and pictorial input intended for human use, such as Srihari's interests in identifying persons in newspaper photographs based on clues in an associated text caption (Srihari, 1994), or Novak and Bulko's (1993) interest in solving textbook Physics problems using the exact text problem statement and associated diagrams that are available to the human reader. These problem assumptions do not allow any automation-friendly modifications in either the diagram or text input, thus preserving the full complexity of the coreference problem, and making the solutions difficult to implement and use.
For example, in previous work, it is assumed that the pictorial input is provided in visual (raster) form, such as a camera image. In such cases, it becomes necessary to perform visual scene interpretation (Ballard et al., 1982) simply to identify the pictorial objects of interest. There can be many ambiguities. In Figure 1, for example, it may be non-trivial to infer that the X's form a pattern to be associated with the rectangular object, or to infer that the label ’ B ’ is attached to the rectangular object,as opposed to the track or to the diagram as a whole.

1.2.1. Our goal

Our goal is to provide a simpler methodology which allows the combined use of text and diagrams to compactly provide the spatial and contextual properties of a scene as input to a given system (Rajagopalan et al., 1994; Rajagopalan, 1994b). This need was forced on us as we pursued another goal, the development of a problem solver for the magnetic fields domain which performs qualitative reasoning about changes in both spatial and nonspatial properties to solve problems (Rajagopalan, 1994a). As we considered how to input a description of the initial state, we found that special purpose text forms were too cumbersome for providing a complete description of the spatial properties, and that diagrams were a far more natural method of providing the same information. However, we still required text input to describe certain non-spatial properties of the initial state, including changes that may be taking place. Thus, we encountered the coreference problem.

1.2.2. Our approach

We have developed a novel solution to the coreference problem based on two observations: (1) Annotations are often used in drawings to allow human readers to associate non-spatial contextual information with objects. For example, Figure 1 contains two means to identify the rectangular object as a magnetic field: the label, B, and the pattern of Xs; (2) Graphical drawing tools provide several methods, such as colors, filling patterns, and brush types, to highlight objects.
Our solution to the problem of compactly providing both spatial and nonspatial information as input to a given reasoning system has been to automate the ability to associate contextual information with the use of highlighting colors, patterns, and brush types for objects in diagrams drawn through graphical drawing tools. We have developed a special description language, the Picture Semantics, for this purpose.

1.2.3. Advantages of our approach

Our approach has two favorable consequences for the coreference problem: (1) The need for additional text input can sometimes be completely eliminated, thus trivially solving the coreference problem; (2) When the need for additional text input persists, our approach allows constraints on both the spatial and contextual properties of objects to be used when matching textual and diagram objects.
Our solution is straightforward to implement and use. The assumption that the pictorial input is given through a graphical editor greatly reduces the complexity of diagram interpretation - anything drawn as a distinct object in the diagram can be readily identified as a distinct object from the output of the graphical editor. Further, properties such as brush types, filling patterns, and colors can be directly associated with objects, thus eliminating a source of ambiguity present in interpreting visual input in raster form.

2. The Picture Semantics description

The diagram and text input to the Figure Understander for the problem of Figure 1 is given in Figure 2. The cruder diagram and greatly simplified text form are sufficient to capture all the necessary input information for the problem. This simplification is possible because of the power of our Picture Semantics description, illustrated in Figures 3 and 4, which allows contextual properties to be associated with the use of brush types, filling patterns, and colors in diagrams.
Incidentally, text labels could also be used in the Picture Semantics if the graphical editor provides a means to directly attach the label to other diagram objects. Otherwise, the task of associating the text label with a diagram object can itself be a challenging problem (He et al., 1994) particularly when the diagram contains overlays. Since the graphical editor we use, the Interviews drawing editor, treats text as a separate object, we shall not consider text labels any further in this paper.

In our approach, we first process the diagram before processing the text input. The result is a partial knowledge base with instantiations for each diagram object under the appropriate knowledge base class. For the diagram in Figure 2, we use the Picture Semantics form to provide the following associations between diagram objects and knowledge base (KB) classes, as illustrated in Figure 3: white objects are conducting disks, gray objects are magnetic fields pointed into the page, black objects are supports, and vectors with the light brush type used in the diagram are motion vectors.
In our application, we always provide a vector in the diagram to provide the direction of motion of objects, intrinsic front directions, and the directions of any forces acting on the objects. In the example of Figure 2, the diagram does not specify which object is in motion, since the vector is floating in free space. This information is obtained from the text input, "the disk is rolling down the track".

2.1. Solving the coreference problem in Figure 2

After processing the text input in Figure 2, we learn that there must be a mobile, conducting disk in the diagram which is in motion (from 'disk is rolling'). Further, we learn that there is contact between the disk and a supporting track (from 'down the track'). From our application-wide assumption described above, we also infer that there must be a motion vector present.
The coreference problem in processing the diagram and text in Figure 2 is to identify the conducting disk, motion vector, and supporting track in the diagram, even though the diagram primitives are only lines, circles, and polygons.
The problem is quickly solved by use of the Picture Semantics description, since, when processing the diagram, we have already extracted the information that there are instances of conducting disks, magnetic fields, supporting objects, and motion vectors in the diagram. Thus, once the text processing identifies the need for finding a conducting disk, motion vector, and supporting track, we can directly search against the knowledge base for diagram objects with these contextual properties.
Since there is only one object of each KB class in the diagram, the matching process becomes straightforward, and we can assert the direction of motion specified by the vector into the KB description of the disk. In more complex scenes, when there are multiple instances of a given object type in the diagram, spatial constraints in the text, such as the clue that the conducting disk and supporting track must be in contact, can be used to isolate the specific instances of interest. In Section 4, we present an urban scene interpretation example where there are multiple instances of each object type.

3. The Figure Understander

The Figure Understander is an implemented research tool for integrating the information in text and graphically drawn diagrammatic input into a single knowledge base description. The intended use of the Figure Understander is to support applications, such as map understanding systems, story understanding systems, or automated problem solving systems, that will benefit from the ability to compactly describe the spatial state of the world through a diagram, and yet may require additional text-based input to describe the domain properties of the diagram objects and any dynamic changes that are taking place.
The input to the Figure Understander includes a diagram produced through an object-oriented drawing editor (currently, the Postscript file for a diagram produced using the Interviews drawing editor), a Picture Semantics description for the diagram, one or more sentences of descriptive text, and an Algernon (Crawford et al., 1991) knowledge base. It outputs an integrated description of the information in each form of input, diagram and text, into the knowledge base.

3.1. Diagram processing

The strength of the Figure Understander lies in its diagram interpretation capabilities. As the diagram is processed, a knowledge base (KB) object is created for each diagram object. Both the contextual information given in the Picture Semantics and the qualitative spatial properties of interest are attached to the KB description of the object. The spatial properties extracted include the spatial extent (size, shape, and position), connectivity, and orientation of individual objects, and topological and directional spatial relations between objects. For example, the connectivity of a polygon is stored by creating knowledge base structures for all the edges and vectors, and storing the connections between the edges and vertices. Incidentally, the Figure Understander also contains heuristics to allow for human errors in drawing the diagrams, such as adjusting for an unintentional gap of a few pixels between objects intended to be touching each other.

3.1.1. The spatial model

The Figure Understander extracts and reasons with a qualitative internal description of the initial spatial state. Our spatial representation, described in greater detail in (Rajagopalan, 1994a), has been used to solve problems involving static scenes as well as problems involving translational and rotational motion. For the purposes of this paper, we will assume that all objects are 2-dimensional and are confined to the XY plane.
Spatial relations between objects are evaluated using 'ideal meanings', as defined by Herskovits (1985). More specifically, for problems involving static scenes or only translational motion, we use a minimum bounding rectangle, similar to Chang, et al., (1987) and Jungert (1992), to approximate the spatial extent of each object. Two objects are assumed to satisfy a spatial relation if their bounding rectangles satisfy the relation.
The minimal bounding rectangle is drawn around the extremal edges or vertices of an object: its rightmost (rm) ,leftmost (lm) ,topmost (tm) ,and bottommost (bm) vertices or edges,as shown in Figure 5. Then,as shown in Figure 6, we define eight regions around the bounding box of each object. The relative positions of two objects are determined based on the regions around the bounding box of a reference object that are intersected by bounding box of the second object.
We model both directional (e.g., right-of, left-of) and topological (e.g., next-to, adjacent, intersecting) spatial relations between objects in terms of qualitative inequality constraints on the X -coordinate values of the rightmost and leftmost extremal edges of their bounding boxes, and the Y-coordinate values of the topmost and bottommost extremal edges of the bounding boxes. The inequality relations that define the relative positions of objects in an input diagram are extracted automatically by the Figure Understander.
The list of binary spatial relations recognized by the Figure Understander is given in Figure 7.

We model the orientation of an object in two ways: (1) the surface normal direction of each distinct surface (edge) of the object, as used by Nielsen (1988), and (2) the angle between the plane of the object and each of the planes formed by the coordinate axes. The latter is required only for reasoning about rotation and will not be discussed further here.
For (1), we can obtain the surface normal direction of any edge or arc of a convex object from knowledge of the identities of the vertices and edges which form the extremal points/surfaces of the object. For example, if an edge or arc lies between the topmost and rightmost extremal points/surfaces, such as edge E1 in Figure 5, then its surface normal direction will have positive X and Y components. We similarly model all 8 qualitative orientations in 2-dimensional space as presented by Freksa (1992) and Mukerjee and Joe (1990).
The distinction in our work is that while the 'idealized' minimum bounding rectangles are used as the basis for describing the relative positions of objects, for reasoning about orientation, we maintain knowledge of the actual orientation of the surfaces of an underlying object. This is necessary to model such properties as the direction of current flow along a particular surface of an object, the directions of magnetic forces acting on an object, and the intrinsic front direction of an object.
Further details of the advantages and limitations of our spatial model, its extensions for reasoning about translational and rotational motion, and details of its implementation within the framework of existing qualitative modeling and simulation systems (Kuipers, 1994) may be found in our AAAI-94 article (Rajagopalan, 1994a).

3.2. Text processing

The task of attaching semantic information given in any accompanying text sentences to diagram objects requires one to first locate the appropriate diagram objects to which the information is to be attached. We require that the text must provide sufficient information to uniquely locate the diagram objects to which the information is to be attached.
The Figure Understander performs text processing after processing the diagram. Since the Picture Semantics provides the conceptual class for each diagram object, if we can obtain the conceptual class of each text object, text and diagram objects could be matched based on their conceptual class. Algemon’s (Crawford et al., 1991) definite description facility ( : the ) is used to connect object references in text to objects in the knowledge base (KB). The form ( : the ?v1 (member conducting-disks ?v1)) asks Algernon to find a unique object, to be bound to the variable ?v1, that is a member of the knowledge base class conducting - disks. If it is the case that there is more than one conducting disk in the knowledge base, we can use any additional constraints, such as spatial relations, obtained from processing the text to narrow the search of the knowledge base.
The text processing is accomplished by first parsing a sentence through an ATN grammar, and then interpreting the parse output through a domain-dependent semantic interpreter. For each supported sentence type, the semantic interpreter examines the subject, verb, direct object, etc., and assembles a query-and-assert form in the syntax of the Algernon language for locating the appropriate diagram objects in the knowledge base (query) and attaching any new information given in the text about the object (assert).
Figure 9 shows the Algemon form generated after processing the sentence 'The disk is rolling down the track'. There are three queries (:the) and one new assertion. The queries search for diagram objects using both spatial properties and contextual properties. The first form, ( : the ?v1 (member conducting-disks ?v1) (mobility-of ?v1 mobile)), is a query to locate a unique diagram object which is a member of the knowledge base (KB) class 'conducting-disks', and which is also a mobile object. The second query is to locate a member of the KB class 'support-objects' such that the conducting disk and support object are touching each other. The third query searches for a motion vector and retrieves its direction. The assert then assigns the direction of the motion vector as the velocity vector of the rolling disk.

4. A scene interpretation example

In Section 2, we discussed the use of the Picture Semantics description to provide contextual information, such as the knowledge base (KB) class for each diagram object, directly within the diagram. We showed that when each diagram object is of a unique KB class,the solution to the coreference problem of matching text and diagram objects becomes straightforward. In this section, we consider solution strategies for the coreference problem when there are multiple objects of the same contextual class in the diagram. Essentially, in such cases, the Figure Understander expects additional constraining information in the text to uniquely identify the appropriate diagram object. One useful source of additional constraints is the use of spatial relations in the text.

4.1. Intrinsic/Deictic frames of reference

Retz-Schmidt (1988) discusses that in human use of spatial prepositions, extrinsic, intrinsic, or deictic frames of reference could be used. The Figure Understander reasons about spatial relations with respect to the extrinsic frame of reference of the diagram. However, to allow reasoning about deictic and intrinsic frames of reference, the Figure Understander's spatial reasoning module allows queries about directional spatial relations to carry a frame of reference argument, and automatically determines an equivalent relation to prove in terms of the extrinsic frame of reference of the diagram. In Figure 10 , the vectors define the front direction for each object. Currently, the Figure Understander only considers intrinsic and deictic frames of reference that are orthogonal to that of the diagram.

5. Discussion

This paper has addressed the question "Are there any simple, but unrestrictive, methods that allow diagrams and text to be used as computer input?". The goal is to take advantage of the power of diagrams for describing a spatial state (Larkin et al., 1987; Narayanan et al., 1994) and yet retain the ability to use text-based input to describe other contextual properties of interest. This goal requires the development of solutions to the coreference problem - matching object references in text to objects in the diagram.
Previous work in this area has resulted in the development of methods for automating the ability to understand text and diagrammatic input intended for human communication (He et al., 1994; Novak et al., 1993; Rowe et al., 1993; Srihari, 1994). This assumption makes the coreference problem particularly complex and difficult for several reasons: (1) There can be hidden assumptions on the background knowledge of the human reader which may be not be available to an automated reasoning system; (2) Even the task of independently understanding each form of input may be difficult. Understanding pictorial input in visual (raster) form requires visual scene interpretation (Ballard et al., 1982), and processing text intended for human-to-human communication can require a sophisticated natural language processing system to resolve the ambiguities that may be present.
We have described a simpler methodology, the Picture Semantics description, and an implemented tool, the Figure Understander, which allows multimedia input to a knowledge base in the form of a graphically-drawn diagram and a collection of text sentences to be integrated into a single unified description.
By using graphical editors to provide diagrammatic input, we gain two advantages over systems that have to deal with visual input. First, graphical editors directly identify the objects of interest by providing a geometric description of the objects in the scene. If the additional constraint that diagram objects must be in one-to-one correspondence with the objects to be created in the knowledge base is enforced, there is no need for visual scene interpretation.
Second, graphical editors allow additional properties to be attached to objects such as patterns, colors, and brush types. Our system uses a special input form, the Picture Semantics, to associate contextual knowledge with these graphical properties. This approach provides a simple and elegant solution to the coreference problem by allowing text and diagram objects to be matched based on both spatial and domain-specific contextual information.

5.1. Applicability of our solution

We impose strict restrictions on the contents of both the diagram and the text. For example, unlike the work of Srihari (1994), we do not consider the possibility that there may be object references in the text for which there is no corresponding object in the diagram. To avoid ambiguity in establishing coreference, we also require that there is at least one unique object in the diagram which can be matched against a text object based on its Picture Semantics description alone. We further assume that it will be possible to use spatial relations in the text between such uniquely identifiable objects and the non-unique diagram objects to isolate any specific non-unique diagram object.
The reader may ask, 'Aren't these requirements too restrictive?'. The answer is yes, if, as with the work of Srihari (1994), Rowe (1993), and Novak (1993), the goal is to automatically process text and diagram input intended for human use, such a problem statement in a textbook, or a newspaper photograph. That is why their solutions to the coreference problem are necessarily more complex.
The solutions provided in this paper are intended for users who simply wish to take advantage of diagrams to compactly input the many spatial properties of a spatial scene, but who need another means, such as text, to provide additional domain-specific information about the objects in the diagram. In these situations, it is quite reasonable to assume that the user has a great interest in providing ambiguity-free input descriptions.
Just as we are making use of the Figure Understander to process diagram and text descriptions of a problem in our work on qualitative reasoning about the effects of magnetic fields (Rajagopalan, 1994a), the work of Hari Narayanan, et al. (1994) and He et al. (1994) could also benefit from the simple approach presented in this paper for using diagrams and text as computer input.
Further, although the focus of this paper has been on our solutions for the coreference problem, the reader may find the diagram processing and spatial reasoning capabilities of the Figure Understander to be worthy of further study. The Figure Understander automatically chooses bounding shape abstractions for objects in diagrams, can choose a frame of reference for a diagram to maximize the alignments of vectors and object surfaces with the coordinate axes, and automatically compensates for simple human errors in drawing a diagram, such as slight misalignments between objects. The Figure Understander also automatically extracts a qualitative description of a spatial scene from an input diagram, including the shapes (if the shape is simple) of objects, the connectivity (in terms of edges and vertices) of polygons, the relative positions of objects, and the orientation of objects.
Although the Figure Understander reasons about a static input scene, we have also developed methods for qualitatively reasoning about simultaneous dynamic changes in both the spatial and non-spatial properties of a physical system. The interested reader may consult (Rajagopalan, 1994a) or (Rajagopalan, 1995) for further details.

5.2. Related issues

This paper has focused on a very specific problem - providing an efficient method for using text and diagrams as computer input. Some related research issues include the generation of explanations using both text and graphical input (Feiner et al., 1990), generating a pictorial description from a text description of a scene (Gapp, 1994; Latecki et al., 1992; Olivier et al., 1994), and generating a text description from a visual image of a scene (Maass, 1994).
The solutions to these issues focus less on associating pictorial objects with text objects, and instead, concentrate on the modeling and use of spatial relations. For example, Gapp (1994), Latecki and Pribbenow (1992), and Olivier, et al., (1994) have all pointed out that the interpretation of distance-based topological relations, such as near and far, and those relating two paths, such as along, can be highly context dependent. They have defined both heuristic and quantitative methods for evaluating the applicability of such topological relations to a set of objects in a given spatial scene.
